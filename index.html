Welcome to BTEC Resources Pages.

Listed here are BTEC Resources sample pages supplied in Adobe Indesign and PDF document format. These have been converted into the equivalent PXE XHTML markup. The basis of the content structure is the HTML5 semantic whereby all content has been broken down and classified as Articles <Article> inside Section <Section> or Aside <Aside> HTML5 semantic components. For each discrete content asset the closest match PXE semantic has been assigned. Where no close match PXE semantic was available we have needed to invent a new PXE semantic that best represents the asset and submitted to PXE community board for wide acceptance.

Here is the outline structure used to assemble the header <header>, footer <footer>, article <article>, Section content chunks within nested sections and aside within the document body <body>

Format of document sections for Content and Learning Objectives:

  • Unit introduction
  • Learning aims
    • Learning aims and unit content
  • Assessment criteria
  • Teacher guidance
    • Resources
    • Assessment guidance
  • Suggested assignment outlines

The process used to markup the specs into PXE is a manual one comprising of these steps:

  1. Create a new PXE asset using the [BTEC Resources PXE Master Asset template - these are articles type which need to be defined].
  2. Update / add meta data and taxonomy tags into meta data. TODO: generate URI of taxonomy tag from EVN.
  3. From sample Indesign or PDF doc, copy the content into the relevant sections under the section headings of HTML5 in the template.
  4. Apply PXE semantic classes to headings, sections, tables. Re-use from PXE Core in the first instance.
  5. Generate bespoke PXE classes if required. Override or add new styles which are not available in PXE Core.
  6. Generate OPF manifest file to link page items together.
  7. Assemble into Epub3 bundling XHTML, CSS, Fonts, Images, Audio, Video and Javascript dependencies.
  8. Validate Epub3 with EpubCheck validator tools. Fix errors and repeat from step 6 until Epub passes.
  9. We have a digital output!
